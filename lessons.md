# Lessons Learned: Knowledge System Recovery (JOB-1538)

## 사건 개요
- **문제**: `knowledge-system-docs` 프로젝트의 핵심 파일들이 유실되어 프로젝트 시스템에서 인식되지 않음.
- **원인**: 배포 작업(gh-pages) 과정에서 스크립트 및 설계서가 정리되었고, 이 상태가 백업본으로 그대로 전파됨.

## 교훈
1. **스크립트 보존의 중요성**: 빌드 결과물뿐만 아니라 생성 스크립트(`generate-slides.py`)와 설계서(`specs/`)는 반드시 `main` 브랜치에 머지되어 보존되어야 함.
2. **Git 히스토리의 신뢰성**: 로컬 백업본이 파일 유실 상태였으나, Git 브랜치(`main`)에는 완벽하게 보존되어 있어 Git이 가장 확실한 백업 수단임 확인.
3. **복구 전 상태 검증**: 복구 작업 시작 전 체크섬(Checksum)을 찍어, 작업이 의도치 않게 다른 파일에 영향을 주지 않았음을 검증하는 절차가 필수적임.
4. **브랜치 역할 명확화**: `main`은 안정적인 코드와 자료의 저장소, `gh-pages`는 빌드된 결과물 전용으로 역할을 명확히 구분해야 함.
