### [SPEC-B7] 슬라이드 구성 규격

## 내용 작성 규칙

### 중복 방지 규칙
- **슬라이드 내**: 본문(개념), 다이어그램(흐름), 팝업(기술 명세) 역할 분리. 중복 금지.
- **슬라이드 간**: 본문↔본문 금지, 다이어그램↔다이어그램 금지. 팝업↔팝업은 공통 개념일 경우 허용.

### 역할 분리 (Role Separation)
| 위치 | 허용 내용 | 금지 내용 |
|------|-----------|-----------|
| **본문** | 핵심 컨셉, 개략적 알고리즘, 문제 정의, 팝업 트리거 키워드 포함 | 다이어그램 상세 요약, 기술 명세 |
| **다이어그램** | 프로세스 흐름, 구조, 순서 시각화 | 본문 내용 반복, 텍스트-heavy |
| **팝업** | **[키워드]: 세부 기술 설명, 수식, 명령어, 각주 (리스트 형식)** | 본문 컨셉 반복, 다이어그램 설명 |

### 팝업 세분화 및 트리거 규칙
- **작성 방식**: `## 팝업` 섹션에 `- **키워드**: 설명` 형태로 작성.
- **트리거 생성**: 본문 내 해당 키워드가 최초로 등장하는 지점에 `?` 아이콘(인라인 트리거) 자동 삽입.
- **동작**: 트리거 클릭 시, 해당 키워드에 매핑된 특정 설명만 팝업창에 출력.
- **중복 제어**: 슬라이드당 동일 키워드의 트리거는 최초 1회만 생성.


### 디자인 및 언어 규칙
- **배경/폰트**: 흰색 배경, h1:56px, h2:44px, 본문:16px
- **언어**: 기술 용어는 영어(Context Bloat, RAG, LLM 등), 나머지는 한국어.
- **다이어그램**: 순차적/분류적 동작 시 필수 사용.

## 검증 기준
- **자동 검증**: 텍스트 유사도 80% 이상 → 경고 (validate-slides.py)
- **수동 검증**: 작성자/리뷰어 확인
- **일관성 검증**: Spec 내용 ↔ HTML 렌더링 결과 일치 여부
