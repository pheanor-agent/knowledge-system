### [SPEC-B7] 슬라이드 구성 규격

## 슬라이드별 내용 구조

### 슬라이드 1: Cover
```
- 라벨: 에이전트 메모리 시스템
- 제목: "AI 에게 기억을 선물하다"
- 부제목: "챗봇에서 파일 기반 지식 시스템으로"
- 설명: "사용자의 입력을 체계적으로 저장하고 활용하는 방법"
- 메타: 날짜, 발표자
```

### 슬라이드 2: 문제 제기
```
- 라벨: 문제
- 제목: "왜 AI 에게 파일이 필요한가요?" + 팝업트리거
- 시나리오: 대화 종료 시 정보 소실 문제
- 질문: "→ 파일을 활용하면 기억을 남길 수 있습니다"
- 팝업: 문제 상세 설명 (본문과 중복 없음)
```

### 슬라이드 3: 신입사원 예시
```
- 라벨: 비유
- 제목: "신입사원이 배우는 3 가지 지식 관리 방법" + 팝업트리거
- 계층 1: 회사 백과사전 외우기 (고정 지식)
- 계층 2: 개인 노트 작성하기 (계속 추가됨) ← 중요
- 계층 3: 참고 자료 책상 위에 준비 (상황별 참조)
- 팝업: 실제 기업 교육 사례
```

### 슬라이드 4: AI 의 3 층 기억 + 연결
```
- 라벨: AI 의 3 층 기억
- 제목: "신입사원이 배우듯, AI 도 3 가지 방식을 활용합니다" + 팝업트리거
- 좌측: 신입사원 3 가지 방법
  - 1: 백과사전 외우기
  - 2: 개인 노트 작성하기 ← 오늘 주제
  - 3: 참고 자료 준비
- 우측: AI 3 가지 방식
  - 1: 모델 학습 (학습한 지식)
  - 2: 에이전트 메모리 (파일로 저장하는 지식) ← 오늘 주제
  - 3: RAG (준비해둔 참고 자료)
- 하단: "오늘은 2 번 '에이전트 메모리' 에 해당하는 파일 기반 지식 시스템을 다룹니다."
- 팝업: 모델 학습, 에이전트 메모리, RAG 기술적 설명
```

### 슬라이드 5: 시스템 개요
```
- 라벨: 시스템 개요
- 제목: "에이전트 메모리 시스템" + 팝업트리거
- 다이어그램: 원본 데이터 → AI 가 정리 → 구조화된 지식
- 통계: 3 단계 파일 정리, 10 개 주제 분류, 3 단계 중요도 레벨
- 팝업: Karpathy 3 층 구조, append-only, metadata.json
```

### 슬라이드 6: 파일 정리 3 단계
```
- 라벨: 3 단계 구조
- 제목: "파일을 정리하는 3 단계" + 팝업트리거
- 다이어그램: 1 단계 원본 → 2 단계 정리 → 3 단계 규칙
- 카드 3 개:
  - 원본 데이터: 대화 기록, 작업 내용 (불변 원본)
  - 정리된 지식: AI 가 요약한 문서 (검색 가능)
  - 분류 규칙: 중요도, 태그 (자동 분류)
- 팝업: Karpathy 3 층 구조, append-only, metadata.json
```

### 슬라이드 7: 자동화 과정
```
- 라벨: 자동화
- 제목: "자동으로 파일을 정리하는 과정" + 팝업트리거
- 다이어그램: 수집 → 분류 → 점수 매기기 → 노출
- 카드: 매일 자동 실행 (대화 기록 수집, 주제별 분류, 중요도 점수 계산, 검색용 인덱스 생성)
- 팝업: sync-all.sh, fetch.sh, 점수 계산 공식
```

### 슬라이드 8: 점수 → 중요도 연결
```
- 라벨: 중요도
- 제목: "점수에 따라 3 단계 중요도로 분류합니다" + 팝업트리거
- 다이어그램: 점수 계산 → 중요도 분류 → 노출 방식
- 카드 3 개:
  - T1 핵심 (0.7 점 이상): 항상 노출
  - T2 일상 (0.4-0.7 점): 상황별 노출
  - T3 참고 (0.4 점 미만): 요청 시 제공
- 팝업: 점수 계산 공식 상세, recency_decay, threshold
```

### 슬라이드 9: 주제 분류
```
- 라벨: 주제 분류
- 제목: "지식을 주제별로 분류하는 체계" + 팝업트리거
- 카드 6 개:
  - 🤖 에이전트: 작업 과정, 규칙, 자동화
  - ⚙️ 시스템: 인프라, 설정, 구성
  - 🧠 메모리: 사용자 선호, 경험 교훈
  - 💬 대화: 대화 기록, 컨텍스트
  - 📚 지식: 연구, 학습 자료
  - ...: 외 5 개 주제
- 팝업: 각 도메인별 상세, 파일 수
```

### 슬라이드 10: 지식 찾는 과정
```
- 라벨: 검색
- 제목: "AI 가 지식을 찾는 과정" + 팝업트리거
- 다이어그램: 진입점 → 목록 → 필터 → 상세
- 카드 4 개:
  - 1 단계 진입점: knowledge-navigation 스킬
  - 2 단계 목록: index.md 파일
  - 3 단계 필터: domain-*.md, tag-*.md
  - 4 단계 상세: pages/*.md
- 팝업: knowledge-navigation, index.md, domain-*.md
```

### 슬라이드 11: Thank You
```
- 라벨: Thank You
- 제목: "세미나 완료"
- 본문: 핵심 참고 자료
  - 📖 지식 중요도 분류: 링크
  - 📖 Karpathy 3 층 구조: 링크
- 메타: 에이전트 메모리 시스템, 날짜, GitHub 링크
```

## 내용 작성 규칙

### 본문 vs 팝업 분리
| 위치 | 내용 | 예시 |
|------|------|------|
| 본문 | 컨셉과 개략적 알고리즘 | "파일을 정리하는 3 단계" |
| 팝업 | 세부 적용 기술 | "Karpathy 3 층 구조, append-only, metadata.json" |

### 다이어그램 사용 규칙
| 동작 유형 | 다이어그램 사용 | 예시 |
|-----------|----------------|------|
| 순차적 동작 | 사용 | 수집 → 분류 → 점수 매기기 → 노출 |
| 분류적 동작 | 사용 | T1/T2/T3 분류 |
| 단순 설명 | 사용하지 않음 | 주제 분류 소개 |

### 언어 규칙
- 기술 용어: 영어 사용 (RAG, LLM, API)
- 나머지: 한글 표기
- 중국어: 사용 금지

### 팁 사용 규칙
- 본문과 중복 내용: 금지
- 뻔한 내용: 금지
- 추가 통찰 또는 실제 사례: 권장
