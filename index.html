문제

왜 AI 에게 파일이 필요한가요?

에이전트에게 무조건 많은 정보를 한 번에 주입하면 부작용이 발생합니다. Context Bloat? 와 Lost in the Middle? 현상이 발생하여 효율성이 급감합니다. 이를 해결하기 위해 물리적인 파일 형태의 선택적 기억 장치가 필요합니다.

비유

신입사원이 배우는 3 가지 지식 관리 방법

계층 1: 회사 백과사전 외우기 (고정 지식) 계층 2: 개인 노트 작성하기 (계속 추가됨) 계층 3: 참고 자료 책상 위에 준비 (상황별 참조) 실제 업무 환경에서 문서를 다루는 방식은 AI 의 데이터 처리 레이어와 일치합니다.

AI 의 3 층 기억

신입사원이 배우듯, AI 도 3 가지 방식을 활용합니다

1. 사전 학습 모델 (Model Training): 변경 불가한 고정 지식. 2. 에이전트 메모리 (File System): 실시간으로 추가되는 개인 메모리. 3. 문서 참조 (Retrieval-Augmented Generation): 외부 연동되는 참조 자료. 오늘은 2 번 '에이전트 메모리' 에 해당하는 파일 기반 지식 시스템을 다룹니다.

시스템 개요

에이전트 메모리 시스템

수집된 경험을 바탕으로 시스템이 스스로 분류 기준을 세우고 탐색 체계를 구축합니다. 단편적인 대화는 구조화된 데이터로 치환되어 영구 보존됩니다.

컴파일 워크플로우

지식을 파일로 저장하는 3 단계 과정

대화가 종료되면 의미 있는 정보를 식별하여 분리하고, 연관성을 찾아 묶어준 뒤, 규칙에 맞게 다듬습니다. Ingest?(지식 추출) → Link?(관계 연결) → Lint?(무결성 검증)

자동화

자동으로 파일을 정리하는 과정

사람의 개입 없이 정해진 스케줄에 따라 파이프라인이 구동되어 최신 상태를 유지합니다. 배치 스크립트(주기적 실행) → 메타데이터(태그 스캔) → 점수화(가중치 갱신)

중요도

동적 점수에 따른 3 단계 분류

T1: 항상 참조해야 하는 핵심 행동 규칙 T2: 특정 상황에 직면했을 때 꺼내보는 일상 지식 T3: 요청이 있을 때만 제공되는 보조 배경 정보 점수 계산과 감가상각 알고리즘을 통해 불필요한 Context 의 메모리 점유를 방지합니다.
