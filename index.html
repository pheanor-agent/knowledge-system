AI 에게 기억을 선물하다

챗봇에서 파일 기반 지식 시스템으로
사용자의 입력을 체계적으로 저장하고 활용하는 방법

문제

왜 AI 에게 파일이 필요한가요??

에이전트에게 무조건 많은 정보를 한 번에 주입하면 부작용이 발생합니다. 자료가 방대해질수록 정작 중요한 지시 사항은 망각하게 되며 효율성이 급감합니다. 이를 해결하기 위해 물리적인 파일 형태의 선택적 기억 장치가 필요합니다.

비유

신입사원이 배우는 3 가지 지식 관리 방법?

계층 1: 회사 백과사전 외우기 (고정 지식) 계층 2: 개인 노트 작성하기 (계속 추가됨) ← 중요 계층 3: 참고 자료 책상 위에 준비 (상황별 참조)

AI 의 3 층 기억

신입사원이 배우듯, AI 도 3 가지 방식을 활용합니다?

좌측: 신입사원 3 가지 방법 - 1: 백과사전 외우기 - 2: 개인 노트 작성하기 ← 오늘 주제 - 3: 참고 자료 준비 우측: AI 3 가지 방식 - 1: 사전 학습 모델 (Model Training) - 2: 에이전트 메모리 (File System) ← 오늘 주제 - 3: 문서 참조 (Retrieval-Augmented Generation) 하단: 오늘은 2 번 '에이전트 메모리' 에 해당하는 파일 기반 지식 시스템을 다룹니다.

시스템 개요

에이전트 메모리 시스템?

수집된 경험을 바탕으로 시스템이 스스로 분류 기준을 세우고 탐색 체계를 구축합니다.

원본 데이터(대화 수집)
→
AI 가공(요약 분류)
→
구조화된 지식(인덱싱)
컴파일 워크플로우

지식을 파일로 저장하는 3 단계 과정?

대화가 종료되면 의미 있는 정보를 식별하여 분리하고, 연관성을 찾아 묶어준 뒤, 규칙에 맞게 다듬습니다.

Ingest(지식 추출)
→
Link(관계 연결)
→
Lint(무결성 검증)
자동화

자동으로 파일을 정리하는 과정?

사람의 개입 없이 정해진 스케줄에 따라 파이프라인이 구동되어 최신 상태를 유지합니다.

배치 스크립트(주기적 실행)
→
메타데이터(태그 스캔)
→
점수화(가중치 갱신)
중요도

동적 점수에 따른 3 단계 분류?

T1: 항상 참조해야 하는 핵심 행동 규칙 T2: 특정 상황에 직면했을 때 꺼내보는 일상 지식 T3: 요청이 있을 때만 제공되는 보조 배경 정보

Score 계산(다중 변수 종합)
→
Tier 배정(T1/T2/T3)
→
Context 로딩(전략적 배치)
주제 분류

지식을 주제별로 분류하는 체계?

특정 도메인에 종속되지 않고 다양한 주제를 수용할 수 있는 카테고리 확장을 지원합니다.

에이전트(작업 프로세스) / 시스템(인프라 설정) / 메모리(선호도 및 교훈) / 지식(연구 자료)
계층적 탐색

AI 가 지식을 찾는 4 단계 과정?

전체 자료를 무분별하게 읽어오지 않고, 지도 역할을 하는 최상위 파일부터 상세 문서까지 단계적으로 좁혀가며 필요한 정보만 정확히 찾아냅니다.

Navigation(탐색 도구 실행)
→
Index Scan(전체 맵 확인)
→
Domain Filter(범위 좁히기)
→
Detail Read(상세 내용 로드)
