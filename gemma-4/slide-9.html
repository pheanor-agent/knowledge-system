주제 분류

지식을 주제별로 분류하는 체계

에이전트 메모리 시스템은 모든 지식을 4 개의 주제 브랜치로 나누어 체계적으로 관리합니다. 물리적 폴더 구조가 아닌 태그 기반 논리적 분류를 통해 유연한 확장을 지원합니다.

📚 지식 시스템 (Knowledge System)
🤖

에이전트 (Agent)

에이전트의 행동 정의

├── 스킬 (Skills)
├── 플러그인 (Plugins)
└── 워크플로우 (Workflows)
⚙️

시스템 (System)

인프라 및 운영 환경

├── 설정 (Config)
├── 인프라 (Infra)
└── Cron (정기 작업)
🧠

메모리 (Memory)

지식과 학습 기록

├── 시스템 메모리
├── 학습 (Learning)
└── 경험 (Experience)
📖

참고 자료 (Reference)

외부 지식 소스

├── 위키 (Wiki)
├── 가이드 (Guide)
└── 레퍼런스
🏷️

Metadata 기반 분류

파일별 metadata.json 이 도메인, 태그, 우선순위, 갱신 일자를 기록하여 물리적 경로를 분리하지 않고도 논리적 주제를 판별합니다. 이 방식은 파일 이동 없이 태그만으로 브랜치를 변경할 수 있어 확장성이 뛰어납니다.

📁

디렉토리 구조

실제 디렉토리 트리는 도메인별로 분리되어 저장되며, Index 파일이 상위에서 하위를 참조합니다. 에이전트는 Index → 도메인 → 파일의 계층 구조를 따라 지식을 탐색하고 로드합니다.

💡Tip: 태그 기반 분류는 물리적 폴더 이동 없이 Metadata 만 수정하면 브랜치 전체가 변경되어 확장 비용이 거의 제로에 가깝습니다. 새 도메인을 추가할 때 기존 구조를 수정할 필요가 없습니다.
